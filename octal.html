What is Octal?

Octal is a base-8 number system that uses 8 digits: 0, 1, 2, 3, 4, 5, 6, 7. It provides a more compact way to represent binary numbers, especially useful in Unix/Linux systems.

üí° Why Octal Matters:
  • File Permissions: chmod 755, 644 commands in Unix/Linux
  • Compact Binary: Each octal digit = exactly 3 binary digits
  • Embedded Systems: Used in low-level programming
  • Historical Importance: Common in older computer systems

Octal in Unix/Linux File Permissions

Understanding chmod with Octal

File permissions use 3 octal digits representing Owner, Group, and Others.

Permission Bits:

Permission | Binary | Octal | Meaning
-----------|--------|-------|------------------
   ---     |  000   |   0   | No permissions
   --x     |  001   |   1   | Execute only
   -w-     |  010   |   2   | Write only
   -wx     |  011   |   3   | Write + Execute
   r--     |  100   |   4   | Read only
   r-x     |  101   |   5   | Read + Execute
   rw-     |  110   |   6   | Read + Write
   rwx     |  111   |   7   | All permissions

Example: chmod 755 explained

7 = 111 = rwx  (Owner: read, write, execute)
5 = 101 = r-x  (Group: read, execute)
5 = 101 = r-x  (Others: read, execute)

Command: chmod 755 myfile.sh
Result: -rwxr-xr-x myfile.sh

Example: chmod 644 explained

6 = 110 = rw-  (Owner: read, write)
4 = 100 = r--  (Group: read only)
4 = 100 = r--  (Others: read only)

Command: chmod 644 document.txt
Result: -rw-r--r-- document.txt

üîß Common chmod Values:

  • 755: Scripts/executables (owner full, others read+execute)
  • 644: Regular files (owner read+write, others read)
  • 600: Private files (owner only)
  • 777: Full access (not recommended - security risk!)
